Why haXe?

As javascript projects get larger, developers need to manage complexity; traditionally, the first tool for this task is the compiler. The haXe compiler generates Javascript (also PHP, C++, SWF) and can help manage complexity.

haXe is a great fusion of strong/weak typing, oop/functional styles, and has a super fast compiler.

  • Type Inference
  • Classes, interfaces, typedefs (multiple defs in one file)
  • Closures
  • Generics
  • Supercharged enums (OCaml like discriminated unions)
  • Lambda library
  • Full reflection
  • Meta Data
  • Macros
  • Easy definition of type signatures for external libraries.

Language ref (pdf) .

It's great to create code incrementally in a dynamic language, in my experience the grief is in refactoring. A typed language makes refactoring large projects much easier by ensuring all variables/structures correspond everywhere.

Switching languages and marshalling data between tiers is a recipe for code duplication and errors. haXe removes this friction by providing typed RPC, and the ability to share data structures between tiers.

haXe can build all tiers incredibly quickly from a simple but effective make file, and you can be sure all the elements of your system are in sync.

Remember also that haXe targets PHP, Flash and mobile too. That means that your team can continue to target all the main vehicles of web development from one language!

Why Node.js?

Node.js is a powerful yet lightweight serverside platform for Windows/Linux. I believe it's low resource utilisation will mean we'll find it everywhere - probably supplanting PHP on shared hosts.

Node.js is the best serverside platform available for haXe and can be used instead of neko in most situtations. It's particularly suitable for high traffic sites when proxied behind another web server, e.g. nginx.

Node.js available for most Unix and on Windows (Microsoft supports Node.js development).

  • Industrial Strength; Google V8, Lars Bak (hotspot compiler)
  • Fast; Javascript compiled directly to machine code.
  • Huge mindshare, Javascript is the defacto language of the web
  • Small Footprint, great for shared hosting environments.
  • Simple, minimalist API.
  • All asynchronous, based on well tested libs.
  • Single threaded, removing a class of potential bugs.
  • Node.js is the best and most robust serverside platform for haXe.
  • Liberally licensed.
